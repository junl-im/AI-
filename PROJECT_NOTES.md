# PROJECT NOTES v1.3.4

## 설계 결정

- PC는 여러 작업 구간과 8개 메뉴를 항상 확인할 수 있게 유지합니다.
- 모바일은 핵심 4개 메뉴를 우선 표시하되 `전체 메뉴`로 기존 8개 기능 접근을 보장합니다.
- 현재 단계와 다음 단계는 모바일 간단 메뉴에서 항상 노출합니다.
- 단계가 바뀌면 전체 메뉴를 자동으로 접어 다음 행동을 다시 선명하게 보여줍니다.
- 사용자 노출 명칭은 `메뉴바`로 통일하고 내부 `dock` 클래스·ID는 호환성을 위해 유지합니다.
- 핵심 UI는 전용 SVG 아이콘만 사용합니다.
- 매 릴리스마다 전체 설치 ZIP과 직전 버전 덮어쓰기 ZIP을 함께 제공합니다.

## 모바일 메뉴 유지 규칙

1. 작업 순서는 `file → recommend → candidates → preview → waveform → cut → edit → export`입니다.
2. 간단 메뉴에는 파일 열기, 현재 단계, 다음 단계와 문맥상 중요한 한 단계를 포함합니다.
3. 현재 단계가 숨겨지는 조합을 만들지 않습니다.
4. 전체 메뉴를 펼치면 8개 항목이 모두 접근 가능해야 합니다.
5. `aria-expanded`, 숨은 탭의 `aria-hidden`, 현재·다음 라이브 안내를 유지합니다.
6. PC에서는 축약 규칙이 메뉴 레이아웃에 영향을 주지 않아야 합니다.
7. 720px 이하에서만 모바일 메뉴 컨트롤을 시각적으로 노출합니다.
8. 모바일 페이지 가로 overflow는 0px를 유지합니다.

## 긴 파일 메모리 원칙

1. metadata 확인 뒤 파일 길이·크기·기기 메모리로 성능 예산을 만듭니다.
2. `estimatedDecodeMemoryMb`, `memoryRisk`, `hardBlock`을 분석 시작 전에 평가합니다.
3. 고위험 파일은 사용자 안내와 `decode-memory-warning` 진단을 남깁니다.
4. 안정 범위를 크게 넘는 파일은 브라우저가 멈추기 전에 변환·분할을 안내하고 중단합니다.
5. `decodeAudioData()` 입력을 위해 원본 ArrayBuffer 전체 복사본을 만들지 않습니다.
6. 분석 전용 트랙은 6–8kHz 모노로 줄이고 UI yield와 AbortSignal을 유지합니다.
7. 분석 완료 후 AudioBuffer와 channelData를 앱 상태에 보관하지 않습니다.
8. 무압축 WAV·AIFF의 디코딩 자체 메모리 한계는 알려진 제한으로 계속 명시합니다.

## 런타임·QA 원칙

- 정적 QA와 실제 Chromium 감사를 모두 통과해야 릴리스합니다.
- PC 8개 메뉴, 모바일 간단 4개와 전체 8개를 각각 검사합니다.
- MP3·MP4 정상 저장뿐 아니라 취소, 실패 재시도, 장시간 분석과 ETA를 검사합니다.
- 출력 파일은 ffprobe로 컨테이너·재생 시간·크기를 확인합니다.
- QA 보고서 수치는 실제 `npm test` 결과와 일치해야 합니다.
