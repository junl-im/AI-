# PROJECT NOTES v1.1.9

## 설계 결정

- 반응형은 단순 축소가 아니라 기기별 작업 방식으로 나눕니다.
- PC의 넓은 화면은 주요 제작 구간을 동시에 보여주는 데 사용합니다.
- PC 첫 작업 프레임은 `불러오기/추천 | 미리보기 | 후보/파형`의 3열 구조입니다.
- 컷, 편집, 저장, 프로젝트는 같은 그리드의 다음 행에 배치해 전체 흐름을 끊지 않습니다.
- 모바일 첫 화면에는 실행 버튼을 중복 배치하지 않습니다. 파일 선택은 Dock이 담당합니다.
- 모바일 본문은 4단계 진행 순서를 가장 먼저 이해시키는 역할만 합니다.
- 상단 메타 정보는 양 끝 정렬, 본문 설명은 제한된 행 길이를 사용합니다.
- 마지막 CSS 레이어에서 레이아웃을 통제해 기존 엔진과 DOM 앵커를 건드리지 않습니다.
- 상태 동기화는 MutationObserver, 사용자 이벤트, 커스텀 흐름 이벤트를 우선합니다.

## 사용자 스타일 키워드

`PC 분할 작업실`, `모바일 단순 진행`, `기능 중복 제거`, `양 끝 정렬`, `넉넉한 텍스트 여백`, `영화적이지만 실용적인 글래스 UI`, `한눈에 다음 행동 파악`
