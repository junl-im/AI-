# PROJECT NOTES - v0.8.1

## 이번 판단

Dock은 모든 기능을 담는 공간이 아니라, 사용자가 제일 자주 누르는 시작 액션을 크게 제공하는 공간으로 재정의했습니다. 이전 7버튼 Dock은 기능은 많았지만 시각적으로 복잡했고, 모바일에서 버튼이 작아지는 문제가 있었습니다.

## 설계 원칙

- Dock은 📂 파일 열기 / ⚡ 분석하기만 제공
- 두 버튼은 50:50 반반 배치
- 이모지와 명확한 한국어 라벨 사용
- 추천/편집/출력 기능은 기존 카드 내부 버튼으로 유지
- 파일 로드 후 레이아웃 붕괴 방지 유지
- 폴링보다 이벤트/RAF 기반 동기화 우선
- 무거운 패널은 CSS containment로 레이아웃 비용 최소화

## v0.8.1 성능 포인트

- Dock polling 제거
- Dock sync RAF 배치
- 미리보기 still 렌더 RAF 배치
- 카드/리스트 containment 추가
- preview/waveform 카드 content-visibility 적용
