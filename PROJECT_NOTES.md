# PROJECT NOTES v1.2.4

## 설계 결정

- 반응형은 단순 축소가 아니라 기기별 작업 방식으로 나눕니다.
- PC 첫 작업 프레임은 `불러오기/추천 | 미리보기 | 후보/파형`의 3열 구조입니다.
- 모바일 첫 화면에는 실행 버튼을 중복 배치하지 않습니다. 파일 선택은 하단 메뉴바가 담당합니다.
- 사용자에게 보이는 하단 내비게이션 명칭은 `메뉴바`로 통일합니다. 내부 클래스·ID·파일명의 `dock`은 호환성을 위해 유지합니다.
- PC 상단은 `제품 카피 | 9:16 쇼츠 비주얼 + 시작 액션`의 2열 구조를 사용합니다.
- PC의 상단 시작 버튼은 새 입력 상태를 만들지 않고 기존 `fileInput`에 연결합니다.
- 920px 이하에서는 PC용 9:16 비주얼과 시작 액션을 제거해 모바일 메뉴바 시작 흐름과 중복되지 않게 합니다.
- 상단 메타는 왼쪽 `BUILD + 버전 + 모바일·PC 호환`, 중앙 로컬·비공개·9:16 상태, 오른쪽 `DESIGNED BY + 곰같은여우`의 한 기준선으로 정렬합니다.
- 상단 스타일은 `hero-command-deck.css`가 단독 소유합니다. 예전 `cinematic-hero.css`를 다시 링크하지 않습니다.
- 상태 동기화는 현재 값과 목표 값이 다를 때만 DOM을 수정합니다.

## 쇼츠 상단 디자인 원칙

1. 쇼츠의 정체성은 휴대폰 그림이 아니라 `9:16 프레임`, `0:00~0:15 타임라인`, `HOOK/BEAT/CAPTION` 같은 실제 제작 언어로 표현합니다.
2. 외곽 프레임은 하나만 사용하고 내부에는 범용 카드 대신 세로 컷 프레임만 둡니다.
3. 큰 제목과 짧은 설명, 한 줄 타임라인이 먼저 읽혀야 합니다.
4. 색상은 검은 바탕 위 시안·바이올렛·핑크의 한 흐름으로 제한합니다.
5. 버전과 디자인 서명은 배지가 아니라 작은 메타 텍스트로 처리합니다.
6. PC 상단의 시작 액션은 하나만 사용합니다.
7. 모바일에서는 세로 프레임 비주얼과 시작 액션을 제거하고 메뉴바의 파일 열기를 유지합니다.
8. 애니메이션은 타임라인의 느린 진행감 하나만 사용하며 모션 감소·저사양 모드에서 중단합니다.
9. 제목 아래 설명은 최대 폭을 유지하고 화면 끝까지 늘리지 않습니다.
10. 쇼츠 느낌을 위해 이모지, 플랫폼 로고, 과한 네온 테두리를 남발하지 않습니다.
11. 메뉴 아이콘은 운영체제별 컬러 이모지 대신 단색 스튜디오 글리프를 사용합니다.
12. PC 진행 단계는 메뉴 활성화만 바꾸지 않고 해당 패널의 위치와 시각적 포커스까지 함께 이동합니다.

## 첫 렌더 최적화 원칙

- 상단 전용 CSS를 여러 파일에서 덮어쓰지 않습니다.
- 첫 화면 밖의 무거운 상세 설정은 `content-visibility`를 기능 감지 후 사용합니다.
- 레이아웃 점프를 막기 위해 `contain-intrinsic-size`를 함께 지정합니다.
- 캔버스 크기나 실측이 필요한 핵심 미리보기·파형에는 무분별하게 `content-visibility`를 적용하지 않습니다.

## 페이지 멈춤 회귀 방지 규칙

1. `textContent`, `innerHTML`, `setAttribute`, `classList`와 `dataset` 쓰기 전에 현재 값을 비교합니다.
2. MutationObserver의 `attributes: true`는 필요한 `attributeFilter`가 있을 때만 사용합니다.
3. 렌더 함수는 동일 상태로 연속 호출되어도 새 Mutation을 만들지 않아야 합니다.
4. 초기화 뒤 RAF와 Mutation 콜백 수가 계속 증가하는지 브라우저 프로브로 확인합니다.
5. 새 UI 모듈은 기존 동기화 모듈과 동일한 DOM 소유권을 중복해서 갖지 않습니다.

## 진행 내비게이션 원칙

- 파일 선택 즉시 분석 화면으로 이동합니다.
- 분석 완료 후 추천이 필요하면 추천, 후보가 생성되면 후보 화면으로 이동합니다.
- 후보 선택 후 미리보기, 렌더 시작 후 저장 화면으로 이동합니다.
- 자동 이동과 수동 메뉴 클릭은 `FlowDirectorFinal`이 단일 소유합니다.
- 레거시 탭 컨트롤러는 자체 클로저의 오래된 `active` 값을 DOM에 쓰지 않고, 동기화 전에 `data-active-flow-tab`을 읽어 단일 소유자의 상태를 채택합니다.
- PC는 모든 패널을 유지하되 현재 대상 카드에 시각적 포커스를 줍니다.
- 모바일은 활성 패널만 보여주고 메뉴의 활성 항목이 가로 영역 중앙에 오도록 합니다.
- 모바일의 일시적 업데이트 알림은 상단 메타 레일을 가리지 않고 하단 메뉴바 위에 표시합니다.
