# PROJECT NOTES v1.2.9

## 설계 결정

- 반응형은 단순 축소가 아니라 기기별 작업 방식으로 나눕니다.
- PC 첫 작업 프레임은 `불러오기/추천 | 미리보기 | 후보/파형`의 3열 구조입니다.
- 모바일 첫 화면에는 실행 버튼을 중복 배치하지 않습니다. 파일 선택은 하단 메뉴바가 담당합니다.
- 사용자 노출 명칭은 `메뉴바`로 통일합니다. 내부 클래스·ID·파일명의 `dock`은 호환성을 위해 유지합니다.
- 상단 메타는 왼쪽 `BUILD + 버전 + 모바일·PC 호환`, 오른쪽 `DESIGNED BY + 곰같은여우`의 두 축만 사용합니다.
- 핵심 UI는 전용 SVG 아이콘을 사용하며 운영체제별 이모지와 문자형 아이콘을 다시 넣지 않습니다.

## PC 메뉴바 레이아웃 원칙

1. 메뉴바 외부 프레임이 너비를 소유하고 내부 `.bottom-dock-tabs`는 항상 `width: 100%`를 사용합니다.
2. 내부 레일에 `100vw` 기반 독립 너비나 부모보다 큰 `min-width`를 두지 않습니다.
3. 1180px 이상에서는 8개 항목 한 줄, 그 미만에서는 4열×2행을 사용합니다.
4. 각 항목은 `minmax(0, 1fr)`로 계산해 라벨 때문에 전체 폭이 늘어나지 않게 합니다.
5. 1180·1280·1366·1920px와 390px를 최소 회귀 검수 행렬로 유지합니다.
6. 모든 메뉴의 사각형이 viewport 내부에 있고 페이지 가로 overflow가 0인지 확인합니다.

## 단계형 UI 로딩 원칙

1. 로딩 단계는 `shell → editing → export` 순서를 유지합니다.
2. 메뉴 키는 DOM과 상태 모델이 사용하는 실제 값과 같아야 합니다. 편집 키는 `edit`이며 `editor`를 사용하지 않습니다.
3. 스크립트 쿼리 버전은 `AIShortsRuntimeConfig.BUILD_KEY`에서 파생하며 특정 과거 버전을 하드코딩하지 않습니다.
4. 동일 URL은 중복 삽입하지 않고, 각 자산은 타임아웃과 실패 진단 경로를 가져야 합니다.
5. 포인터·포커스·클릭 의도에서 선로딩하되 사용자의 첫 화면을 막지 않습니다.
6. 로딩 실패는 `data-hydration-error`와 진단 이벤트에 남기고 조용히 삼키지 않습니다.

## 런타임·엔진 안정성 원칙

1. `window.error`와 `unhandledrejection`을 런타임 건강 정보에 기록합니다.
2. 제거한 함수·DOM·상태 키는 이벤트 리스너와 공개 API에 참조가 남지 않았는지 전체 검색합니다.
3. 미디어 `play()`가 거절되면 RAF·interval·렌더 루프를 시작하지 않습니다.
4. RAF, interval, timeout, MediaStreamTrack은 성공·실패·취소 모든 종료 경로에서 정리합니다.
5. 세션 자동 저장은 문서가 보일 때만 heartbeat를 유지하고 숨김 또는 pagehide에서 즉시 정리합니다.
6. DOM 쓰기 전에 현재 값을 비교하고 동일 상태로 Mutation을 반복하지 않습니다.
7. 브라우저 감사는 2초와 4초 샘플의 RAF·Mutation 수가 같고 오류가 0인지 확인합니다.
8. 정적 QA 결과만으로 릴리스를 확정하지 않고 실제 브라우저 스크립트 실행 감사를 함께 보관합니다.

## UI·디자인 유지 원칙

- 멋있지만 깔끔하고 쇼츠의 9:16·빠른 컷·타임라인 정체성이 첫눈에 보여야 합니다.
- 게임 UI처럼 카드·배지·테두리·네온이 겹치면 안 됩니다.
- 바깥 프레임은 한 번만 쓰고 내부 구분은 여백·명암·얇은 선으로 처리합니다.
- 제목 → 현재 작업 → 주요 액션 → 세부 설정 순으로 읽혀야 합니다.
- PC는 여러 작업 패널을 한눈에 보여주고 진행 단계에 맞춰 메뉴와 화면 포커스가 함께 이동해야 합니다.
- 모바일은 기능을 중복 노출하지 않고 하단 메뉴바와 간결한 진행 안내를 사용합니다.
- 활성 아이콘만 강조하고 모든 아이콘에 상시 글로우를 사용하지 않습니다.

## 진행 내비게이션 원칙

- 파일 선택 즉시 추천/분석 작업 영역으로 이동합니다.
- 추천 생성 후 후보, 후보 선택 후 미리보기, 렌더 시작 후 저장 영역으로 이동합니다.
- 자동 이동과 수동 메뉴 클릭은 `FlowDirectorFinal`과 `FlowCommandBridge`를 통해야 합니다.
- PC는 모든 패널을 유지하되 현재 대상 카드에 시각적 포커스를 줍니다.
- 모바일은 활성 작업을 우선 보여주고 메뉴 활성 상태가 함께 이동해야 합니다.

## 배포 원칙

- 매 버전 전체 설치용 ZIP과 직전 버전 덮어쓰기용 패치 ZIP을 함께 제공합니다.
- 패치 ZIP은 `PATCH_MANIFEST.txt`에 기록된 파일만 포함하고 `from/to`를 정확히 적습니다.
- 패치를 깨끗한 이전 버전 복사본에 적용한 뒤 매니페스트 파일이 새 버전과 바이트 단위로 같은지 확인합니다.
- 두 ZIP 모두 `unzip -t`와 SHA-256 체크섬을 제공합니다.
- QA 보고서 수치와 실제 `npm test` 결과가 일치해야 합니다.
