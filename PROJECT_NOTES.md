# PROJECT NOTES - v0.8.2

## 이번 방향

- UI/UX 편의성 강화
- 브랜드 시그니처 추가
- 하단 Dock 단순성 유지
- 버튼 클릭 손맛 개선
- 상태별 햅틱/토스트 피드백 구분

## UX 기준

1. 하단 Dock은 두 개만 둔다.
   - 📂 파일 열기
   - ⚡ 분석하기

2. 반복 작업 버튼은 본문 내부에 둔다.
   - 미리보기
   - 썸네일
   - 내보내기
   - 프로젝트 저장
   - 자막 적용

3. 햅틱은 보조 피드백이다.
   - 지원하지 않는 브라우저에서도 앱은 동일하게 동작해야 한다.
   - 경고/오류는 진동 패턴을 더 길게 둔다.
   - 일반 클릭은 짧게 둔다.

4. 토스트는 유형별로 구분한다.
   - 성공: ✅
   - 액션/분석: ⚡
   - 파일: 📂
   - 내보내기: ⬇️
   - 복사: 📋
   - 경고: ⚠️
   - 오류: 🚫

## 성능 메모

- v0.8.1의 RAF 기반 Dock 동기화 유지
- `setInterval` 폴링 없음
- 무거운 카드 영역 CSS containment 유지
- 피드백 스크립트는 전역 이벤트 위임 방식이라 버튼마다 개별 리스너를 대량 생성하지 않음

v0.8.3에서는 사용자가 켜고 끌 수 있는 `햅틱 ON/OFF` 토글, 토스트 위치 선택, 모바일 한손 모드 등을 고려할 수 있습니다.
