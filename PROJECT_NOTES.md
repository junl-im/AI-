# PROJECT NOTES v1.3.2

## 설계 결정

- PC는 여러 작업 구간을 한 화면에 유지하고 현재 단계만 네온 라인으로 명확하게 강조합니다.
- 모바일은 기능을 중복 배치하지 않고 하단 메뉴바와 현재 패널을 중심으로 진행합니다.
- 사용자 노출 명칭은 `메뉴바`로 통일합니다. 내부 클래스·ID의 `dock`은 호환성을 위해 유지합니다.
- 상단 메타는 왼쪽 BUILD·버전·모바일/PC 호환, 오른쪽 DESIGNED BY·곰같은여우만 사용합니다.
- 핵심 UI는 전용 SVG 아이콘을 사용하며 운영체제별 이모지와 문자형 아이콘을 다시 넣지 않습니다.
- 매 버전 전체 설치용 ZIP과 직전 버전 기준 덮어쓰기 패치 ZIP을 함께 제공합니다.

## 실미디어·오디오 분석 원칙

1. Worker와 메인 스레드는 `audio-analysis-core.js`의 동일 알고리즘을 공유합니다.
2. Worker 실패는 폴백 가능한 장애이며 곧바로 전체 분석 실패로 취급하지 않습니다.
3. 메인 스레드 폴백은 배치 처리·실행권 양보·AbortSignal 확인을 유지합니다.
4. 분석 완료 결과를 쓰기 전에 operation과 media session이 현재인지 확인합니다.
5. Worker 폴백과 기능 누락은 runtime diagnostics에 남깁니다.

## 렌더·복구 원칙

1. MediaRecorder와 canvas captureStream은 렌더 필수 기능입니다.
2. 원본 `play()`가 성공한 뒤에만 recorder를 시작합니다.
3. 원본 재생 실패 시 정지 프레임 녹화를 계속하지 않습니다.
4. 취소는 다운로드를 만들지 않고 `cancelled` 상태로 종료합니다.
5. 실패 재시도는 반드시 새 operation token과 새 queued job으로 실행합니다.
6. 재시도 횟수와 오류 메시지는 job에 유지하되 이전 abort signal을 재사용하지 않습니다.
7. 모든 종료 경로에서 RAF·타이머·MediaStreamTrack·MediaRecorder·볼륨을 정리합니다.
8. 다운로드 직전에도 작업 세대가 현재인지 검사합니다.

## PC 작업실·네온 원칙

1. `workspace-layout-controls.css`가 Prime 작업실의 최종 열 구조를 소유합니다.
2. 저장하는 값은 열 비율뿐이며 집중 보기 모드는 새 실행에 복원하지 않습니다.
3. 현재 단계 강조 최종 소유자는 `FlowDirectorFinal`입니다.
4. 랜딩 애니메이션은 실제 단계 변경 시 한 번만 실행합니다.
5. 집중 보기에서도 메뉴 활성·패널 위치·네온 단계가 일치해야 합니다.
6. 1180px 미만에서는 PC 전용 조절 UI를 숨깁니다.

## 런타임·QA 원칙

- `window.error`와 `unhandledrejection`을 런타임 건강 정보에 기록합니다.
- DOM 쓰기 전에 현재 값을 비교하고 동일 상태 Mutation을 반복하지 않습니다.
- 정적 QA와 실제 Chromium 실행 감사를 모두 통과해야 릴리스합니다.
- MP3·MP4 정상 저장뿐 아니라 취소와 실패 재시도도 실미디어 감사에 포함합니다.
- 출력 파일은 다운로드 발생만 보지 않고 ffprobe로 재생 시간과 크기를 검증합니다.
- QA 보고서의 통과 수치는 실제 `npm test` 결과와 일치해야 합니다.

## 배포 원칙

- v1.3.2 패치 기준은 v1.3.1입니다.
- `PATCH_MANIFEST.txt`는 실제 파일 바이트 차이로 생성합니다.
- 깨끗한 v1.3.1 복사본에 패치를 적용해 매니페스트 파일을 바이트 단위로 비교합니다.
- 두 ZIP 모두 `unzip -t`와 SHA-256 체크섬을 제공합니다.
- 삭제 파일이 생기면 별도 삭제 목록과 적용 절차를 제공합니다.
