# PROJECT NOTES v1.3.1

## 설계 결정

- PC는 여러 작업 구간을 한 화면에 유지하고 현재 단계만 네온 라인으로 명확하게 강조합니다.
- 모바일은 기능을 중복 배치하지 않고 하단 메뉴바와 현재 패널을 중심으로 진행합니다.
- 사용자 노출 명칭은 `메뉴바`로 통일합니다. 내부 클래스·ID의 `dock`은 호환성을 위해 유지합니다.
- 상단 메타는 왼쪽 BUILD·버전·모바일/PC 호환, 오른쪽 DESIGNED BY·곰같은여우만 사용합니다.
- 핵심 UI는 전용 SVG 아이콘을 사용하며 운영체제별 이모지와 문자형 아이콘을 다시 넣지 않습니다.

## PC 작업실 레이아웃 원칙

1. `workspace-layout-controls.css`가 Prime 작업실의 최종 열 구조를 소유합니다.
2. 기본 배치는 왼쪽·가운데·오른쪽 3열과 두 개의 8px 리사이저로 구성합니다.
3. 각 열의 최소 폭은 왼쪽 260px, 가운데 350px, 오른쪽 300px입니다.
4. 포인터 드래그는 `setPointerCapture()`로 소유하고 종료 시 반드시 해제합니다.
5. 키보드는 좌우 방향키와 Home 초기화를 지원합니다.
6. 저장하는 값은 열 비율뿐입니다. 집중 보기 모드는 새 실행에 복원하지 않습니다.
7. `미리보기 집중`과 `파형 확대`는 PC에서만 사용합니다.
8. 집중 보기와 맞지 않는 메뉴를 명시적으로 선택하면 3열 균형으로 복귀합니다.
9. 1180px 미만에서는 모든 작업실 조절 UI를 숨기고 기존 반응형 흐름을 사용합니다.
10. 리사이즈 중 DOM 전체를 재생성하지 않고 CSS 변수만 갱신합니다.

## 현재 단계 네온 랜딩 원칙

1. 자동 진행과 수동 메뉴 클릭은 모두 `FlowDirectorFinal`을 통합니다.
2. 현재 패널에는 `is-navigation-target`과 `is-stage-current`를 유지합니다.
3. 새 단계 진입 시에만 `is-stage-landing`과 `is-navigation-pulse`를 약 1초 적용합니다.
4. `.stage-neon-rail`은 지속 테두리, `.stage-progress-chip`은 현재 단계 명칭을 담당합니다.
5. 집중 보기에서도 현재 단계 강조와 메뉴 활성은 같은 단계 키를 사용합니다.
6. 모션 감소 설정에서는 애니메이션 없이 고정 테두리만 유지합니다.

## 비동기 작업 소유권 원칙

1. 분석·미리보기·렌더는 `AIShortsOperationCoordinator`의 채널을 사용합니다.
2. 새 원본 파일은 새 미디어 세션을 만들고 이전 작업을 취소합니다.
3. 비동기 결과를 상태에 쓰기 직전에 현재 작업인지 검사합니다.
4. 취소는 사용자 오류로 표시하지 않고 진단에 `cancelled`로 기록합니다.
5. 성공·실패·취소는 동일한 cleanup 경로에서 RAF·interval·timeout·MediaStreamTrack·볼륨을 정리합니다.

## PC 메뉴바 레이아웃 원칙

- 외부 `.bottom-dock`이 너비를 소유합니다.
- 내부 `.bottom-dock-tabs`는 `width:100%; max-width:100%; min-width:0`을 유지합니다.
- 1180px 이상은 8개 한 줄, 미만은 4열×2행입니다.
- 1180·1280·1366·1920px와 모바일 390px를 회귀 검수 행렬로 유지합니다.

## 런타임·엔진 안정성 원칙

- `window.error`와 `unhandledrejection`을 런타임 건강 정보에 기록합니다.
- DOM 쓰기 전에 현재 값을 비교하고 동일 상태 Mutation을 반복하지 않습니다.
- 정적 QA와 실제 Chromium 실행 감사를 모두 통과해야 릴리스합니다.
- 단계 전환 후 두 샘플 구간의 RAF·Mutation 수가 같아야 합니다.
- QA 보고서의 통과 수치는 실제 `npm test` 결과와 일치해야 합니다.

## 배포 원칙

- 매 버전 전체 설치용 ZIP과 직전 버전 덮어쓰기용 패치 ZIP을 함께 제공합니다.
- v1.3.1 패치 기준은 v1.3.0입니다.
- `PATCH_MANIFEST.txt`에 변경·추가 파일을 모두 기록합니다.
- 깨끗한 v1.3.0 복사본에 패치를 적용해 매니페스트 파일을 바이트 단위로 비교합니다.
- 두 ZIP 모두 `unzip -t`와 SHA-256 체크섬을 제공합니다.
