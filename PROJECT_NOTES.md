# PROJECT NOTES - AI Shorts Studio v0.4.1

## 패치 의도

사용자가 보고한 문제는 기능 부족이 아니라 파일을 불러온 뒤 UI가 흔들리는 안정성 문제였다. v0.4.1은 새 기능 확장보다 먼저 레이아웃을 고정하고, 반복 작업 버튼을 하단 Dock에 모아 실제 제작 흐름을 정리하는 패치다.

## 원인 가설

v0.4.0에서는 파일 로드 후 `sourceVideo` 또는 `sourceAudio`가 `.is-visible` 상태가 되면서 미리보기 카드 내부에 원본 컨트롤이 표시된다. 이 컨트롤은 브라우저/파일 종류에 따라 높이가 달라지고, 특히 비디오 파일은 메타데이터 로드 후 카드 높이를 밀어내면서 전체 그리드가 틀어져 보일 수 있다.

## v0.4.1에서 바꾼 것

- 원본 미디어 엘리먼트를 화면 밖 1px 처리로 숨김
- 캔버스 미리보기 중심 UX로 정리
- 상단 히어로를 컴팩트 상태로 변경
- 파일 로드 후 `body.has-media` 클래스로 더 압축된 상태 적용
- 기존 상단 action dock은 숨김
- 새 하단 fixed Dock 생성
- Dock 버튼은 원본 버튼을 mirror/click하는 방식으로 안전하게 연결
- 추천/편집 버튼은 각 영역으로 부드럽게 스크롤
- 추천 리스트와 좌측 패널에 스크롤 안정성 적용
- 레거시 모바일 액션바는 DOM 앵커만 유지하고 숨김

## 하단 Dock 기능 배치 이유

하단 Dock은 “제작 중 손이 가장 많이 가는 액션” 순서로 배치했다.

```text
파일 → 분석 → 추천 → 편집 → 미리보기 → 썸네일 → 내보내기
```

- `파일`: 새 원본을 빠르게 교체
- `분석`: 추천 생성의 시작점
- `추천`: 후보 리스트로 이동
- `편집`: 구간/자막/썸네일 설정으로 이동
- `미리보기`: 선택 구간 확인
- `썸네일`: 쇼츠 게시 전 필수 결과물 저장
- `내보내기`: 최종 결과 저장

## 유지한 것

- v0.4.0의 파형 드래그, 썸네일 템플릿, 후보 일괄 내보내기
- v0.3.0의 빠른 길이 칩, 단계 표시, 모바일 대응
- v0.2.0의 자막/프로젝트 저장
- v0.1.0의 무료 로컬 분석 원칙

## 일부러 하지 않은 것

- 원본 video/audio 컨트롤을 사용자에게 다시 노출하지 않았다. 컨트롤이 레이아웃을 다시 흔들 수 있기 때문이다.
- 하단 Dock에 모든 기능을 넣지 않았다. 너무 많은 버튼을 넣으면 모바일에서 복잡해지기 때문이다.
- 자막 세부 스타일 패치는 다음 버전으로 미뤘다.
