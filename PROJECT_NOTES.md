# PROJECT NOTES v1.1.8

## 설계 결정

- 기능을 더 붙이기 전에 첫 화면의 행동 우선순위와 런타임 비용을 정리합니다.
- 시작 화면은 실제로 누를 수 있는 CTA와 현재 제작 단계만 보여줍니다.
- 파일을 연 뒤에는 시작 안내를 접고 현재 작업 패널에 집중합니다.
- 상태 동기화는 주기적 폴링보다 MutationObserver, 사용자 이벤트, 커스텀 흐름 이벤트를 우선합니다.
- 파형 선택 오버레이는 상태 시그니처가 바뀐 경우에만 DOM을 교체합니다.
- 저사양 대응은 기능을 끄지 않고 블러, 그림자, 장식 애니메이션부터 낮춥니다.
