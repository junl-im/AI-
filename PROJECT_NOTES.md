# PROJECT NOTES v1.3.5

## 설계 결정

- 서비스워커 등록은 앱 본체가 아니라 전용 부팅 모듈이 단일 소유합니다.
- 외부에서 들어온 문자열은 기본적으로 `textContent`, `createTextNode`, `replaceChildren`로 출력합니다.
- 오디오·영상으로 판별되지 않는 파일은 드래그 입력에서도 즉시 거부합니다.
- PC 8개 메뉴와 모바일 핵심 4개/전체 8개 흐름은 유지합니다.
- 긴 파일 메모리 사전 점검과 원본 ArrayBuffer 중복 복사 방지 규칙은 유지합니다.
- 매 릴리스마다 전체 설치 ZIP과 직전 버전 덮어쓰기 ZIP을 함께 제공합니다.

## 서비스워커 유지 규칙

1. `src/boot/service-worker-registration.js`가 등록 호출의 최종 소유자입니다.
2. 등록 Promise를 공유해 초기화 중 중복 호출을 막습니다.
3. 성공 시 현재 버전과 scope를 diagnostic에 남깁니다.
4. 실패 시 오류를 처리하고 Promise 캐시를 비워 재시도를 허용합니다.
5. `file:` 환경과 service worker 미지원 환경은 정상적인 unsupported 결과로 처리합니다.
6. HTML 로드 순서와 서비스워커 셸 캐시에 등록 모듈을 함께 반영합니다.

## 사용자 데이터 출력 원칙

1. 파일명과 프로젝트 JSON의 모든 문자열은 신뢰하지 않습니다.
2. 후보 제목·구간, 세션 파일명, 렌더 라벨·오류를 동적 HTML 템플릿에 직접 삽입하지 않습니다.
3. 고정 아이콘·레이아웃 골격만 정적 `innerHTML`을 허용합니다.
4. HTML 문자열 조립이 필요한 기존 경로는 `escapeHtml`을 사용하거나 DOM 생성 방식으로 전환합니다.
5. 회귀 검사에 악성 마크업 형태 문자열을 포함합니다.

## 런타임·QA 원칙

- 정적 QA와 실제 Chromium 감사를 모두 통과해야 릴리스합니다.
- 서비스워커 성공·실패·중복 방지·재시도는 독립 단위 검사로 확인합니다.
- MP3·MP4 정상 저장뿐 아니라 취소, 실패 재시도, 장시간 분석과 ETA를 검사합니다.
- 출력 파일은 ffprobe로 컨테이너·재생 시간·크기를 확인합니다.
- QA 보고서 수치는 실제 `npm test` 결과와 일치해야 합니다.

## 배포 원칙

- v1.3.5 패치 기준은 v1.3.4입니다.
- `PATCH_MANIFEST.txt`는 실제 변경·추가 파일로 생성합니다.
- 전체·패치 ZIP 모두 `unzip -t`와 SHA-256 체크섬을 제공합니다.
