# HANDOFF v1.2.1

## 요약

v1.2.1은 상단 첫인상을 전면 개편한 시네마틱 컨트롤 데크 패치입니다. PC에서는 제품 소개와 `STUDIO READY` 시작 패널을 같은 프레임에 배치했고, 모바일에서는 시작 패널을 제거해 기존 하단 메뉴바와 기능이 중복되지 않도록 했습니다. v1.2.0에서 해결한 MutationObserver 피드백 루프와 응답성 안정화 규칙은 그대로 유지합니다.

## 이번 상단 개편

- 버전은 왼쪽 끝, `Design by 곰같은여우`는 오른쪽 끝 정렬을 유지했습니다.
- 중앙 메타에는 로컬 처리와 PC·모바일 호환 정보를 배치했습니다.
- 제품명 위에 `LOCAL AI CREATIVE SUITE · SHORT-FORM WORKSTATION` 제품 성격을 짧게 표시합니다.
- 제목은 강한 대형 타이포그래피와 느린 스펙트럼 흐름으로 강조했습니다.
- 설명은 화면 전체 폭을 채우지 않고 읽기 좋은 최대 폭을 유지합니다.
- 로컬 분석, 자동 추천, 9:16 렌더를 기능 칩으로 표시합니다.
- 오른쪽 상태 데크에는 `READY`, 필름 릴, 제작 단계와 새 프로젝트 시작 버튼을 배치했습니다.
- 시작 버튼은 새 로직 없이 기존 `fileInput`에 연결된 label입니다.
- 런타임에서 생성되는 YouTube·Reels·TikTok 스트립은 상단 하단의 얇은 출력 포맷 레일로 정리했습니다.
- 860px 이하에서는 상태 데크와 플랫폼 레일을 숨깁니다.
- 720px 이하에서는 타이포그래피와 설명만 압축하고, 파일 열기는 하단 메뉴바가 전담합니다.

## 사용자 디자인 선호

- 사용자가 처음 사이트를 열었을 때 “와, 이거 멋지다”는 인상을 주는 강한 첫 화면을 선호합니다.
- 멋짐은 무작정 장식을 늘리는 방식보다 대형 타이포그래피, 정교한 정렬, 빛과 어둠의 대비, 분명한 행동 버튼에서 나와야 합니다.
- PC와 모바일을 하나의 레이아웃으로 억지로 맞추지 않습니다.
- PC는 넓은 화면을 활용해 입력, 추천, 후보, 미리보기와 파형이 한눈에 들어오는 분할 작업실을 선호합니다.
- 모바일은 기능을 중복 노출하지 않고, 하단 메뉴바를 실행 수단으로 사용하며 본문은 진행 순서를 쉽게 보여줍니다.
- 개발자에게 익숙한 전문 용어보다 일반 사용자가 바로 이해하는 표현을 우선합니다. 사용자 노출 명칭은 `Dock`이 아니라 `메뉴바`입니다.
- 정보는 많아도 시선의 시작점과 다음 행동이 명확해야 합니다.
- 상단 메타 정보는 버전 왼쪽 끝, `Design by 곰같은여우` 오른쪽 끝처럼 정렬 기준이 분명해야 합니다.
- 제목 아래 설명은 화면 폭을 가득 채우지 않고 읽기 좋은 최대 폭과 좌우 여백을 둡니다.
- 영화·카메라 분위기의 글래스 표현은 유지하되 실제 작업 정보와 행동을 침범하지 않게 절제합니다.

## 유지해야 할 동작

- 1180px 이상에서는 `data-desktop-layout="prime"`의 3열 작업실이 적용되어야 합니다.
- PC에서 불러오기, 추천, 미리보기, 후보와 파형 패널은 활성 메뉴와 무관하게 동시에 보여야 합니다.
- PC 메뉴바 버튼은 기존 상태·비활성 규칙을 유지하며 해당 카드로 이동해야 합니다.
- PC 상단 `새 프로젝트 시작`은 반드시 기존 `fileInput`을 사용해야 합니다.
- 860px 이하에서는 `.hero-command-deck`과 `.social-shutter-strip`이 표시되지 않아야 합니다.
- 720px 이하이고 파일이 없는 `file` 단계에서는 본문 불러오기 패널을 표시하지 않습니다.
- 모바일 파일 열기는 하단 메뉴바의 `label[for="fileInput"]`가 담당합니다.
- 모바일 진행 순서는 불러오기 → 자동 분석 → 편집 → 내보내기 2x2 카드로 유지합니다.
- 파일이 로드되면 `body.has-media`에 의해 모바일 진행 안내가 접혀야 합니다.
- 사용자에게 보이는 설명, 접근성 라벨과 도움말에서 `Dock` 용어를 다시 사용하지 않습니다.
- `MutationObserver` 콜백 안에서 같은 텍스트·속성·클래스를 무조건 다시 쓰지 않습니다.
- 관찰 대상에 자신이 변경하는 속성을 포함해야 한다면 반드시 값 비교 또는 상태 시그니처 가드를 둡니다.
- 500~700ms 상시 폴링을 다시 추가하지 않습니다.
- 버전, 빌드 키와 서비스워커 캐시 키는 항상 함께 올립니다.

## 알려진 제한

- PC Prime 작업실의 칼럼 폭은 현재 고정 비율이며 사용자가 직접 드래그해 조절할 수 없습니다.
- 1180px 미만 태블릿은 기존 단계형 작업 흐름을 유지합니다.
- 상단 플랫폼 레일은 런타임 스크립트가 생성하므로 JavaScript가 차단되면 표시되지 않습니다.
- 원본 미디어 파일은 브라우저 보안 정책상 프로젝트 JSON에 포함되지 않습니다.
- 모바일 Safari 및 일부 인앱 브라우저에서는 MediaRecorder 출력 형식이 제한될 수 있습니다.
- 실제 대용량 MP4를 사용한 장시간 렌더 성능은 자동 QA가 아니라 브라우저 실기기 검증이 필요합니다.
