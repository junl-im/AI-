# HANDOFF v1.1.9

## 요약

v1.1.9은 PC와 모바일을 같은 탭 화면으로 축소·확대하던 방식을 분리한 레이아웃 패치입니다. PC는 여러 핵심 구간을 동시에 보는 Prime 작업실이고, 모바일은 중복 불러오기 화면을 제거한 진행 순서 중심 시작 화면입니다.

## 사용자 디자인 선호

- PC와 모바일을 하나의 레이아웃으로 억지로 맞추지 않습니다.
- PC는 넓은 화면을 활용해 입력, 추천, 후보, 미리보기, 파형이 한눈에 들어오는 분할 작업실을 선호합니다.
- 모바일은 같은 기능을 중복 노출하지 않고, 하단 Dock을 실행 수단으로 사용하며 본문은 진행 순서를 쉽게 보여줍니다.
- 정보는 많아도 시선의 시작점과 다음 행동이 명확해야 합니다.
- 상단 메타 정보는 버전 왼쪽 끝, `Design by 곰같은여우` 오른쪽 끝처럼 정렬 기준이 분명해야 합니다.
- 제목 아래 설명은 화면 폭을 가득 채우지 않고 읽기 좋은 최대 폭과 좌우 여백을 둡니다.
- 영화/카메라 분위기의 글래스 표현은 유지하되 실제 작업 공간을 침범하지 않게 절제합니다.

## 유지해야 할 동작

- 1180px 이상에서 `data-desktop-layout="prime"`의 3열 작업실이 적용되어야 합니다.
- PC에서 불러오기, 추천, 미리보기, 후보, 파형 패널은 활성 탭과 무관하게 동시에 보여야 합니다.
- PC Dock 버튼은 기존 상태/비활성 규칙을 유지하며 해당 카드로 이동해야 합니다.
- 720px 이하이고 파일이 없는 `file` 단계에서는 본문 불러오기 패널을 표시하지 않습니다.
- 모바일 파일 열기는 하단 Dock의 `label[for="fileInput"]`가 담당합니다.
- 모바일 진행 순서는 불러오기 → 자동 분석 → 편집 → 내보내기 2x2 카드로 유지합니다.
- 파일이 로드되면 `body.has-media`에 의해 모바일 진행 안내가 접혀야 합니다.
- v1.1.8에서 제거한 500~700ms 상시 폴링을 다시 추가하지 않습니다.
- 버전, 빌드 키, 서비스워커 캐시 키는 항상 함께 올립니다.

## 알려진 제한

- PC Prime 작업실의 칼럼 폭은 현재 고정 비율이며 사용자가 직접 드래그해 조절할 수 없습니다.
- 1180px 미만 태블릿은 기존 탭형 작업 흐름을 유지합니다.
- 원본 미디어 파일은 브라우저 보안 정책상 프로젝트 JSON에 포함되지 않습니다.
- 모바일 Safari 및 일부 인앱 브라우저에서는 MediaRecorder 출력 형식이 제한될 수 있습니다.
- 실제 대용량 MP4를 사용한 장시간 렌더 성능은 자동 QA가 아니라 브라우저 실기기 검증이 필요합니다.
