# HANDOFF v1.6.9 Direct Preview Crop Editor

## 현재 상태
- 앱 버전: `1.6.9`
- build key: `1.6.9-direct-crop-editor`
- 프로젝트 스키마: `5`
- 미리보기 직접 크롭은 편집 단계에서만 지연 적재한다.
- 수동 직접 편집은 화자·얼굴·모션 자동 추적보다 우선한다.

## 핵심 변경
- 미리보기 canvas의 포인터 드래그 기반 X/Y 크롭 이동
- 휠·트랙패드·두 포인터 핀치 확대
- 방향키·Shift+방향키·확대/축소 키보드 조작
- 조작 종료 후 현재 재생 위치 키프레임 자동 저장
- 키프레임 경로·현재 위치 미니맵과 직전 조정 되돌리기
- 기존 스마트 리프레임 키프레임·프로젝트 저장 계약 재사용
- 미리보기·재생·썸네일·최종 렌더의 동일 크롭 결과 유지

## 검수 순서
1. `npm run check` 또는 등록 검사 구간 실행
2. 직접 크롭 브라우저 감사
3. 4개 화면 Chromium 감사
4. 실제 30분 1080p 스마트 크롭 감사
5. 서비스워커 lifecycle·offline 검사
6. 통합본과 덮어쓰기 패치 SHA-256 비교

## 알려진 제한
- 직접 크롭 편집은 스마트 추적 결과가 준비된 편집 단계에서 활성화된다.
- 브라우저 기본 확대 제스처와 충돌하지 않도록 편집 모드에서만 휠·핀치를 점유한다.
- 전체 15→30→15분 미디어 수명주기와 구조 우선순위 CSS는 변경 경로가 아니므로 검증된 이전 근거를 상속하고, 직접 크롭과 30분 1080p 경로는 새로 실측한다.
