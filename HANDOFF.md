# HANDOFF - AI Shorts Studio v0.4.1

## 현재 상태

- 정적 웹앱으로 실행 가능
- 외부 CDN 의존성 없음
- 파일 로드 후 레이아웃 붕괴 방지 패치 적용
- 원본 audio/video 컨트롤은 화면 레이아웃에서 제거하고 백그라운드 처리용으로 유지
- 상단 히어로 영역 압축
- 긴 설명문/장식 미터/불필요 배지 제거
- 기존 상단 작업 도크 숨김
- 새 하단 고정 Dock 추가
- 하단 Dock에서 파일, 분석, 추천 이동, 편집 이동, 미리보기, 썸네일, 내보내기 실행 가능
- 모바일 기존 액션바는 레거시 앵커만 유지하고 새 Dock으로 통합
- 오디오/비디오 파일 로컬 분석 가능
- 추천 카드 및 파형 표시 가능
- 추천 구간 시작/끝 수동 조절 가능
- 파형 위 드래그 핸들로 구간 조절 가능
- 현재 재생 위치로 시작점 맞춤 가능
- SRT/VTT 자막 업로드/붙여넣기 가능
- 자막 스타일 및 싱크 보정 가능
- 썸네일 템플릿 4종 선택 가능
- 9:16 canvas preview 가능
- 썸네일 PNG 저장 가능
- 추천 후보 일괄 내보내기 가능
- 프로젝트 JSON 저장/불러오기 가능
- MediaRecorder 기반 export 가능
- QA smoke test 31/31 통과

## 시작 방법

```bash
npm run serve
```

브라우저에서 `http://localhost:8080` 접속.

## 검수 순서

1. 첫 화면 상단이 이전보다 컴팩트하게 보이는지 확인
2. 하단 고정 Dock이 데스크톱과 모바일 폭에서 모두 보이는지 확인
3. 하단 Dock의 `파일` 버튼으로 MP3, WAV 또는 MP4 파일 업로드
4. 파일 로드 후 원본 audio/video 컨트롤이 미리보기 카드 아래로 튀어나오지 않는지 확인
5. 파일 로드 후 상단 영역이 더 압축되어 세로 공간을 덜 차지하는지 확인
6. 좌측 추천/설정 패널이 화면 높이를 넘을 때 내부 스크롤로 처리되는지 확인
7. 하단 Dock의 상태 문구가 파일명 또는 선택 추천 구간으로 바뀌는지 확인
8. 하단 Dock의 `분석` 버튼이 본문 `분석하고 추천받기`와 같은 동작을 하는지 확인
9. 추천 카드 3개 이상 표시 확인
10. 하단 Dock의 `추천` 버튼이 추천 카드 영역으로 이동하는지 확인
11. 추천 카드 클릭 시 파형 하이라이트와 제목/해시태그 갱신 확인
12. 하단 Dock의 `편집` 버튼이 구간/자막 편집 영역으로 이동하는지 확인
13. 파형 위 파란 선택 구간과 양쪽 핸들이 표시되는지 확인
14. 선택 구간을 드래그해서 전체 위치가 이동하는지 확인
15. 왼쪽/오른쪽 핸들로 시작/끝 시간이 바뀌고 자동 적용되는지 확인
16. `현재 재생 위치로 시작 맞춤` 버튼이 선택 구간 시작점을 이동시키는지 확인
17. 시작/끝 시간을 수정하고 `구간 적용` 클릭
18. 썸네일 템플릿 4종을 바꿀 때 미리보기 디자인이 바뀌는지 확인
19. SRT/VTT 파일 업로드 또는 자막 텍스트 붙여넣기 후 `자막 적용` 클릭
20. 하단 Dock의 `미리보기` 버튼이 본문 미리보기 버튼과 같은 동작을 하는지 확인
21. 하단 Dock의 `썸네일` 버튼으로 선택 템플릿이 반영된 PNG 다운로드 확인
22. 하단 Dock의 `내보내기` 버튼이 본문 내보내기 버튼과 같은 동작을 하는지 확인
23. `후보 일괄 내보내기`로 상위 3개 또는 5개가 순차 저장되는지 확인
24. `프로젝트 저장`으로 JSON 다운로드 확인
25. `프로젝트 불러오기`로 추천/자막/설정/썸네일 템플릿 복원 확인
26. `진단 복사` 버튼으로 JSON 복사 확인

## 알려진 제한

- 브라우저에 따라 MP4 대신 WebM으로 저장된다.
- iOS Safari에서는 MediaRecorder와 captureStream 지원이 제한될 수 있다.
- 매우 긴 4K 영상은 브라우저 메모리 한계로 분석/내보내기가 느릴 수 있다.
- 비디오의 오디오 트랙 디코딩은 브라우저 코덱 지원에 따라 실패할 수 있다. 실패 시 비디오 움직임 분석 중심으로 추천한다.
- 프로젝트 JSON에는 원본 미디어가 포함되지 않는다.
- 자막 자동 생성은 음성인식이 아니라 텍스트 분할 fallback이다.
- 여러 후보 일괄 내보내기는 브라우저의 다중 다운로드 허용 설정에 영향을 받는다.
- 원본 미디어 컨트롤은 의도적으로 숨겼다. 사용자 조작은 캔버스 미리보기와 하단 Dock을 기준으로 한다.

## 배포 메모

- GitHub Pages에 그대로 올릴 수 있다.
- Firebase Hosting에도 정적 파일로 배포 가능하다.
- `.nojekyll` 포함.
- Service Worker 캐시 버전은 `sw.js`, `index.html`, README의 cache key를 같이 갱신한다.
- 새 스크립트를 추가하면 `sw.js`의 `SHELL_FILES`에도 추가해야 한다.
- v0.4.1에서 추가된 `assets/css/layout-dock.css`, `src/ui/bottom-dock.js`는 Service Worker 캐시에 포함되어 있다.
- 기존 `mobileActionBar` DOM 앵커는 QA와 호환성을 위해 남겼지만 CSS에서 숨긴다.
