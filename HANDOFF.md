# HANDOFF v1.2.2

## 요약

v1.2.2는 v1.2.1 상단의 촌스러운 배지·중첩 테두리·영화 장식을 걷어내고, 고급 편집 도구에 가까운 에디토리얼 마스트헤드로 다시 정리한 패치입니다. 버전과 `Design by 곰같은여우`는 같은 기준선의 텍스트 메타 정보가 되었고, PC 시작 영역은 별도 카드가 아닌 작업실에 연결된 세로 레일이 되었습니다. 첫 렌더 다이어트의 시작으로 상단 CSS 두 파일을 하나로 통합하고 첫 화면 밖 상세 패널의 페인트를 지연했습니다.

## 이번 변경

- 왼쪽 메타: `BUILD v1.2.2 · EDITORIAL STUDIO`
- 중앙 메타: `LOCAL · PRIVATE · READY`
- 오른쪽 메타: `DESIGNED BY 곰같은여우`
- 버전과 서명에서 배경, 둥근 테두리, 이모지와 글래스 배지를 제거했습니다.
- 필름 스트립, 뷰파인더 코너, 회전 릴과 제목 밑 스펙트럼 선을 제거했습니다.
- 제품명과 설명은 왼쪽, 시작 레일은 오른쪽에 두되 내부 중첩 카드를 만들지 않습니다.
- 시작 레일은 `WORKSPACE / 01`, 작업 설명, 4단계 흐름과 밝은 단일 시작 버튼으로 구성합니다.
- 시작 버튼은 반드시 기존 `label[for="fileInput"]`을 사용합니다.
- 모바일에서는 중앙 상태와 PC 시작 레일을 숨기지만 BUILD와 DESIGNED BY는 유지합니다.
- 런타임 생성 플랫폼 레일은 세 개의 카드가 아니라 한 줄짜리 에디토리얼 푸터로 표시합니다.
- `cinematic-hero.css`는 더 이상 index와 서비스워커에서 로드하지 않습니다.
- 상단 전용 스타일은 `assets/css/hero-command-deck.css` 하나로 통합했습니다.
- CSS 요청은 39개에서 38개로 감소했습니다.
- `.caption-pro-panel`, `.quality-panel`, `.batch-export-panel`, `.project-card`, `.copy-card`에 기능 감지형 `content-visibility: auto`를 적용했습니다.

## 사용자 디자인 선호

- 첫 화면에서 “와, 멋지다”는 인상을 원하지만 장식이 많은 게임 UI나 값싼 SF 대시보드처럼 보이면 안 됩니다.
- 둥근 배지, 테두리 카드와 글래스 박스를 반복하는 디자인을 선호하지 않습니다.
- 외곽 프레임 하나, 정교한 기준선, 큰 제목, 절제된 색과 넉넉한 여백을 선호합니다.
- 버전과 디자인 서명은 화면 양 끝에 정렬하되 서로 다른 부품처럼 튀지 않아야 합니다.
- PC와 모바일을 하나의 레이아웃으로 억지로 맞추지 않습니다.
- PC는 넓은 화면에서 입력, 추천, 후보, 미리보기와 파형이 한눈에 들어오는 분할 작업실을 선호합니다.
- 모바일은 기능을 중복 노출하지 않고 하단 메뉴바를 실행 수단으로 사용하며 본문은 진행 순서를 쉽게 보여줍니다.
- 사용자 노출 명칭은 `Dock`이 아니라 `메뉴바`입니다.
- 제목 아래 설명은 화면 폭을 가득 채우지 않고 읽기 좋은 최대 폭을 유지합니다.

## 유지해야 할 동작

- 1180px 이상에서는 `data-desktop-layout="prime"`의 3열 작업실이 적용되어야 합니다.
- PC에서 불러오기, 추천, 미리보기, 후보와 파형 패널은 활성 메뉴와 무관하게 동시에 보여야 합니다.
- PC 메뉴바 버튼은 기존 상태·비활성 규칙을 유지하며 해당 카드로 이동해야 합니다.
- PC 상단 `새 프로젝트 시작`은 반드시 기존 `fileInput`을 사용해야 합니다.
- 860px 이하에서는 `.hero-command-deck`과 `.social-shutter-strip`이 표시되지 않아야 합니다.
- 720px 이하이고 파일이 없는 `file` 단계에서는 본문 불러오기 패널을 표시하지 않습니다.
- 모바일 파일 열기는 하단 메뉴바의 `label[for="fileInput"]`가 담당합니다.
- 모바일 진행 순서는 불러오기 → 자동 분석 → 편집 → 내보내기 2x2 카드로 유지합니다.
- 파일이 로드되면 `body.has-media`에 의해 모바일 진행 안내가 접혀야 합니다.
- 사용자에게 보이는 설명과 접근성 라벨에서 `Dock` 용어를 다시 사용하지 않습니다.
- 상단에 `cinematic-hero.css`를 다시 링크하지 않습니다.
- 상단 내부에 별도 둥근 상태 카드와 장식용 릴을 다시 만들지 않습니다.
- `MutationObserver` 콜백 안에서 같은 텍스트·속성·클래스를 무조건 다시 쓰지 않습니다.
- 500~700ms 상시 폴링을 다시 추가하지 않습니다.
- 버전, 빌드 키와 서비스워커 캐시 키는 항상 함께 올립니다.

## 알려진 제한

- 초기 CSS는 38개로 줄었지만 스크립트 54개는 아직 모두 초기 로드됩니다.
- `content-visibility`는 지원 브라우저에서만 적용되고 미지원 브라우저는 기존 방식으로 렌더합니다.
- PC Prime 작업실의 칼럼 폭은 고정 비율이며 사용자가 직접 드래그해 조절할 수 없습니다.
- 1180px 미만 태블릿은 기존 단계형 작업 흐름을 유지합니다.
- 원본 미디어 파일은 브라우저 보안 정책상 프로젝트 JSON에 포함되지 않습니다.
- 모바일 Safari 및 일부 인앱 브라우저에서는 MediaRecorder 출력 형식이 제한될 수 있습니다.
- 실제 대용량 MP4를 사용한 장시간 렌더 성능은 자동 QA가 아니라 브라우저 실기기 검증이 필요합니다.
