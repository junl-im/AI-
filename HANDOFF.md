# HANDOFF v1.1.8

## 요약

v1.1.8은 기능 추가보다 초기 UX와 상시 런타임 비용을 정리한 첫 기반 패치입니다. 기존 8단계 흐름과 엔진 구조는 유지하면서 첫 화면을 단순화하고, 불필요한 폴링과 DOM 재생성을 제거했습니다.

## 이번 패치의 핵심 파일

- `assets/css/foundation-polish.css`
- `src/ui/startup-performance.js`
- `src/ui/ux-controls.js`
- `src/ui/range-drag-controls.js`
- `qa/runtime_performance_smoke.js`
- `index.html`
- `sw.js`

## 유지해야 할 동작

- 시작 화면에서 파일 열기와 프로젝트 불러오기 버튼이 보여야 합니다.
- 시작 화면의 제작 흐름은 불러오기 → 자동 분석 → 편집 → 내보내기 순서입니다.
- 파일이 로드되면 `body.has-media`에 의해 시작 패널이 접혀야 합니다.
- 하단 Dock은 파일 열기 / 추천 / 후보 / 미리보기 / 파형 / 컷 / 편집 / 저장 순서를 유지합니다.
- 화면 이동의 최종 소유자는 계속 `flow-director-final.js`와 `flow-command-bridge.js`입니다.
- UX와 파형 드래그 모듈에 500~700ms 상시 폴링을 다시 추가하지 않습니다.

## 검수 순서

1. `npm run check`
2. PC에서 첫 화면 CTA와 4단계 레일 확인
3. 모바일에서 CTA가 한 열, 레일이 2x2로 표시되는지 확인
4. 파일 로드 후 시작 패널이 사라지고 추천 탭으로 연결되는지 확인
5. 파형 구간 드래그, 숫자 입력, 스냅 버튼이 정상 반영되는지 확인
6. 저사양 프로필에서 장식 애니메이션과 블러가 줄어드는지 확인
7. 실제 미디어로 추천 → 미리보기 → 저장 E2E 확인

## 알려진 제한

- 원본 미디어 파일은 브라우저 보안 정책상 세션에 자동 저장하지 않습니다.
- 모바일 Safari 및 일부 인앱 브라우저에서는 MediaRecorder 형식이 제한될 수 있습니다.
- v1.1.8은 요청 수 번들링보다 런타임 DOM 비용과 화면 계층을 먼저 개선한 버전입니다.

## 기존 기반 유지

- `Update Sentinel`의 버전/캐시 진단과 이전 캐시 정리 흐름은 그대로 유지합니다.
