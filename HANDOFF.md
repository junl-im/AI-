# HANDOFF v1.2.4

## 요약

v1.2.4는 PC Prime 작업실에서 메뉴바가 단순 활성 표시만 바꾸고 실제 작업 화면을 충분히 안내하지 못하던 문제를 수정한 **Navigation Focus 패치**입니다. 파일을 열면 분석 화면, 분석이 끝나면 추천 또는 후보 화면, 후보를 고르면 미리보기, 렌더를 시작하면 저장 화면으로 메뉴와 작업 카드가 함께 이동합니다. 상단 버전 옆에는 PC·모바일 공통으로 `모바일 · PC 호환` 정보를 표시하고, 메뉴의 컬러 이모지는 단색 스튜디오 글리프로 통일했습니다.

## 이번 변경

- `src/app.js`의 진행 이동은 `AIShortsFlowCommandBridge` → `AIShortsFlowDirectorFinal` → 기존 탭 API 순서로 단일 내비게이션 소유자를 우선합니다.
- 실제 브라우저에서 발견된 레거시 `hyperflow-tabs`의 로컬 `active=file` 덮어쓰기를 제거했습니다. 동기화 시 `data-active-flow-tab`을 먼저 읽어 단일 소유자의 상태를 채택합니다.
- 파일 자동 분석 시작 시 `recommend` 패널을 즉시 표시하고, 분석 완료 후 기존 로직에 따라 `recommend` 또는 `candidates`로 이어집니다.
- 후보 카드 선택은 `preview`, 렌더 시작 액션은 `export` 화면으로 이동합니다.
- PC Prime 작업실에서 현재 대상 패널에 외곽 강조, `현재 작업` 레이블과 한 번의 포커스 펄스를 표시합니다.
- 활성 메뉴에는 `aria-current=step`을 적용하고, 가로 메뉴에서 활성 항목을 중앙으로 가져옵니다.
- 상단 버전 옆에 CSS로 그린 PC·모바일 기기 기호와 `모바일 · PC 호환` 문구를 배치했습니다.
- 중앙 상태는 `LOCAL · PRIVATE · 9:16`으로 정리했습니다.
- 메뉴 글리프는 `＋ 파일`, `✦ 추천`, `◆ 후보`, `▶ 미리보기`, `∿ 파형`, `✂ 컷`, `◫ 편집`, `↓ 저장`으로 통일했습니다.
- 렌더 큐와 보조 이동 버튼의 이모지도 같은 문법으로 정리했습니다. 파형·저장·업데이트 상태에 남아 있던 컬러 이모지도 단색 기호로 교체했습니다.
- 모바일 업데이트 알림을 상단에서 하단 메뉴바 위로 이동해 버전·호환 정보와 디자인 서명을 가리지 않게 했습니다.
- Chromium에서 합성 20초 WAV로 `파일 선택 → 추천 → 후보 → 미리보기` 실제 진행을 검증했습니다.
- 새 회귀 검사 `qa/navigation_focus_smoke.js`를 추가했습니다.

## 사용자 디자인 선호

- 멋있지만 깔끔하고, 쇼츠의 9:16·빠른 컷·타임라인 정체성이 보여야 합니다.
- 게임 UI나 값싼 SF 대시보드처럼 카드·배지·테두리가 중첩되면 안 됩니다.
- 버전, 호환 정보와 디자인 서명은 같은 기준선의 작은 메타 텍스트로 보여야 합니다.
- PC와 모바일을 억지로 같은 화면으로 만들지 않습니다.
- PC는 여러 작업 패널이 한눈에 보여야 하며 메뉴바가 현재 작업 위치를 분명히 안내해야 합니다.
- 모바일은 실행 기능을 중복 노출하지 않고 하단 메뉴바를 사용합니다.
- 사용자 노출 명칭은 `Dock`이 아니라 `메뉴바`입니다.
- 운영체제마다 모양이 달라지는 컬러 이모지보다 의미가 일정한 단색 아이콘을 선호합니다.

## 유지해야 할 동작

- 1180px 이상에서는 3열 Prime 작업실의 모든 주요 패널이 동시에 보여야 합니다.
- 자동 진행과 수동 메뉴 클릭은 모두 `FlowDirectorFinal`을 통해야 합니다. 레거시 모듈은 자체 로컬 탭 값을 DOM에 다시 쓰지 말고 현재 `data-active-flow-tab`을 채택해야 합니다.
- 현재 패널 강조는 작업 기능을 가리거나 레이아웃 크기를 바꾸면 안 됩니다.
- 모바일에서 PC용 9:16 시작 액션은 숨기고 파일 열기는 하단 메뉴바가 담당합니다.
- 상단 `새 쇼츠 만들기`는 기존 `fileInput`을 사용합니다.
- 버전 옆 `모바일 · PC 호환` 문구는 PC와 모바일 모두 표시해야 합니다.
- 중앙 상태 문구를 다른 모듈이 이모지 기반 문구로 덮어쓰면 안 됩니다.
- 메뉴 아이콘은 `＋ ✦ ◆ ▶ ∿ ✂ ◫ ↓` 기준을 유지합니다.
- 같은 DOM 상태를 반복해서 쓰는 MutationObserver 피드백 루프를 다시 만들지 않습니다.
- 500~700ms 상시 폴링을 추가하지 않습니다.
- 버전, 빌드 키와 서비스워커 캐시는 함께 올립니다.

## 검수 순서

1. `npm run check`
2. PC에서 파일을 선택하면 추천/분석 패널이 강조되고 메뉴의 추천 항목이 활성화되는지 확인
3. 분석 완료 후 추천 화면 또는 후보 화면으로 자동 이동하는지 확인
4. 후보 카드를 누르면 미리보기 패널과 메뉴가 함께 활성화되는지 확인
5. 저장을 시작하면 내보내기 패널로 이동하는지 확인
6. 메뉴를 직접 누르면 대상 패널이 강조되고 활성 메뉴가 가로 영역 중앙에 보이는지 확인
7. 1600px PC에서 버전 바로 옆에 모바일·PC 호환 정보가 보이는지 확인
8. 390px 모바일에서도 버전과 호환 정보가 겹치지 않는지 확인
9. 메뉴 글리프가 컬러 이모지로 바뀌지 않고 단색으로 보이는지 확인
10. 모션 감소 설정에서 패널 펄스가 중단되는지 확인
11. 합성 WAV로 파일 → 추천 → 후보 → 미리보기에서 `data-active-flow-tab`, `aria-current`, 대상 패널이 같은지 확인
12. 초기화 및 단계 이동이 끝난 뒤 RAF·Mutation 수가 계속 증가하지 않는지 확인
13. 실제 MP3·MP4로 분석 → 추천 → 후보 → 미리보기 → 저장 E2E 확인
