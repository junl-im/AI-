# HANDOFF v1.2.6

## 요약

v1.2.6은 v1.2.5의 단계형 로딩 구조를 유지하면서 **PC·모바일 전체 UI 문법을 하나로 통일한 디자인 정돈 패치**입니다. 기능 카드마다 반복되던 강한 테두리, 중첩 배경, 글로우를 줄이고 바깥 프레임·작업 패널·입력·버튼·메뉴바의 계층을 공통 토큰으로 재정의했습니다. 분석·추천·렌더 엔진과 진행 내비게이션 로직은 변경하지 않았습니다.

## 이번 변경

- `assets/css/ui-refinement.css`를 최종 캐스케이드 레이어로 추가했습니다.
- 카드와 작업 패널의 테두리·배경·그림자·모서리 값을 공통 디자인 토큰으로 통일했습니다.
- 패널 제목부는 얇은 구분선과 일정한 간격만 사용하도록 정리했습니다.
- 입력, 선택, 텍스트 영역의 높이와 포커스 표현을 통일했습니다.
- 기본 버튼은 차분한 다크 표면, 핵심 버튼은 시안·바이올렛·핑크의 밝은 한 흐름으로 정리했습니다.
- 업로드 영역, 후보 카드, 파형·타임라인의 중첩 프레임과 과한 그림자를 줄였습니다.
- PC 메뉴바를 화면 하단 중앙의 8개 항목 단일 플로팅 레일로 변경했습니다.
- 모바일 메뉴바는 4열×2행을 유지하되 높이, 대비, 아이콘·라벨 간격을 조정했습니다.
- 모바일 상단과 4단계 안내 패널 높이를 줄이고 안내 문장을 짧게 다듬었습니다.
- 비활성 모바일 메뉴도 읽을 수 있도록 대비를 높였습니다.
- 스크롤바를 얇고 일관된 형태로 통일했습니다.
- 저사양 모드와 모션 감소 설정에서는 블러·전환을 계속 축소합니다.
- 버전, 빌드 키, 서비스워커 캐시를 v1.2.6 / `1.2.6-staged-hydration`로 동기화했습니다.

## 사용자 디자인·UX 선호

- 멋있지만 깔끔하고, 쇼츠의 9:16·빠른 컷·타임라인 정체성이 첫눈에 보여야 합니다.
- 게임 UI나 값싼 SF 대시보드처럼 카드·배지·테두리가 겹치면 안 됩니다.
- 바깥 프레임은 한 번만 쓰고, 내부 구분은 여백·명암·얇은 선으로 처리합니다.
- 한 화면에서 모든 요소가 같은 중요도로 보이면 안 됩니다. 제목 → 현재 작업 → 주요 액션 → 세부 설정 순으로 읽혀야 합니다.
- 버전, 모바일·PC 호환 정보와 디자인 서명은 작은 메타 텍스트로 같은 기준선에 둡니다.
- PC는 여러 작업 패널을 한눈에 보여주고, 진행 단계에 맞춰 메뉴와 화면 포커스가 함께 이동해야 합니다.
- 모바일은 기능을 중복 노출하지 않고 하단 메뉴바와 간결한 진행 안내를 사용합니다.
- 사용자 노출 명칭은 `Dock`이 아니라 `메뉴바`입니다.
- 운영체제마다 달라지는 컬러 이모지보다 의미가 일정한 단색 글리프를 사용합니다.
- 최적화는 기능 제거가 아니라 지금 필요한 기능만 먼저 준비하는 방식이어야 합니다.

## UI 유지 규칙

1. 새로운 UI는 먼저 `ui-refinement.css`의 토큰과 표면 규칙을 사용합니다.
2. 기능별 CSS에서 새 색상·반경·그림자를 임의로 추가하지 않습니다.
3. 카드 안에 또 다른 강한 테두리 카드를 중첩하지 않습니다.
4. 현재 작업 강조는 강한 네온 글로우 대신 얇은 시안 테두리와 작은 상태 라벨만 사용합니다.
5. PC 메뉴바는 1180px 이상에서 8개 항목 한 줄을 유지합니다.
6. 모바일 메뉴바는 4열×2행을 유지하고, 파일 열기·현재 단계의 대비를 가장 높게 둡니다.
7. 모바일 첫 화면에는 파일 열기 버튼을 본문에 중복 추가하지 않습니다.
8. 모바일 소개 문장은 고아 단어가 생기지 않게 짧게 유지하고 `text-wrap: balance`를 활용합니다.
9. 기능 아이콘은 `＋ ✦ ◆ ▶ ∿ ✂ ◫ ↓` 단색 글리프 체계를 유지합니다.
10. 저사양·모션 감소 환경에서 사용성을 해치는 블러나 반복 애니메이션을 추가하지 않습니다.

## 단계 로딩 유지 규칙

1. 지연 모듈을 다시 개별 `<script defer>`로 추가하지 않습니다.
2. 로딩 순서는 `shell → editing → export`를 유지합니다.
3. 이미 준비된 단계와 스크립트를 중복 실행하지 않습니다.
4. 포인터·포커스 시점의 사전 준비를 유지해 클릭 후 대기를 줄입니다.
5. 지연 모듈 전체를 서비스워커 설치 프리캐시에 다시 넣지 않습니다.
6. 단계 로딩 후 Observer·RAF가 같은 상태를 반복해서 쓰지 않는지 확인합니다.

## 유지해야 할 기존 동작

- 1180px 이상 PC에서는 3열 Prime 작업실의 주요 패널이 동시에 보여야 합니다.
- 자동 진행과 수동 메뉴 클릭은 `FlowDirectorFinal`과 `FlowCommandBridge`를 통해야 합니다.
- 파일 선택 → 추천, 추천 생성 → 후보, 후보 선택 → 미리보기, 렌더 시작 → 저장 화면 이동을 유지합니다.
- 모바일에서 PC용 9:16 시작 액션은 숨기고 파일 열기는 하단 메뉴바가 담당합니다.
- 버전 옆 `모바일 · PC 호환` 문구는 PC와 모바일 모두 표시합니다.
- 같은 텍스트·속성·클래스를 반복해서 쓰는 Observer 피드백 루프를 다시 만들지 않습니다.
- 버전, 빌드 키와 서비스워커 캐시는 함께 올립니다.

## 알려진 제한 사항

- CSS는 새 최종 레이어를 포함해 39개가 초기 로드됩니다. 다음 최적화에서는 핵심·편집·저장 CSS 분리가 필요합니다.
- 분석·추천·렌더 엔진은 아직 핵심 부팅에 포함되어 직접 실행 스크립트가 39개입니다.

- 실제 대용량 MP4 장시간 렌더, 모바일 Safari와 인앱 브라우저 출력 형식은 실기기 검증이 필요합니다.
