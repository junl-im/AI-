# AI 쇼츠 제작 스튜디오 v1.2.0

음악이나 영상을 브라우저 안에서 분석하고, 쇼츠 후보 추천부터 세로 미리보기와 저장까지 이어주는 로컬 웹 스튜디오입니다.

## v1.2.0 핵심 패치

- 사용자에게 보이는 `Dock` 표현을 이해하기 쉬운 **메뉴바**로 통일했습니다.
- 사이트를 연 직후 발생하던 메인 스레드 점유와 `페이지 응답 없음` 현상을 재현하고 원인을 수정했습니다.
- 추천 개수, 후보 목록, 메뉴 상태를 감시하는 모듈들이 같은 DOM 값을 반복해서 쓰던 피드백 루프를 제거했습니다.
- 상태가 달라진 경우에만 텍스트, 속성, 클래스와 활성 메뉴를 갱신합니다.
- 후보와 작업 공간 감시 범위를 필요한 DOM 변경으로 제한했습니다.
- PC 3열 Prime 작업실과 모바일 4단계 진행 화면은 그대로 유지합니다.
- 버전, 빌드 키와 서비스워커 캐시를 v1.2.0으로 함께 갱신했습니다.

## 화면별 사용 흐름

### PC

첫 화면의 분할 작업실에서 원본을 열고 추천 설정, 후보 목록, 세로 미리보기와 파형을 함께 확인합니다. 하단 메뉴바는 해당 작업 카드로 빠르게 이동하는 내비게이션으로 동작합니다.

### 모바일

첫 화면에서는 제작 순서만 확인합니다. 하단 메뉴바의 `📂 파일 열기`를 누르면 파일 선택기가 바로 열리고, 이후 현재 단계의 패널만 표시됩니다.

## 실행

```bash
npm run serve
```

브라우저에서 `http://localhost:8080`을 엽니다. 파일 탐색기에서 `index.html`을 직접 여는 것보다 로컬 서버 실행을 권장합니다.

## 검수

```bash
npm run check
```

v1.2.0 기준 문법, DOM 앵커, 렌더·추천·저장 모듈, PC/모바일 레이아웃과 런타임 피드백 루프 방지까지 103개 검사를 실행합니다.

## 업데이트 확인

상단 버전 배지를 누르면 현재 버전, 빌드 키, 서비스워커, 캐시와 엔진 프로필을 확인할 수 있습니다. 이전 캐시가 남아 있으면 정보 창에서 이전 캐시 정리를 실행한 뒤 새로고침합니다.

## 모듈형 엔진

분석, 추천, 컷 감지, 렌더 품질, 캐시와 안정성 감사 모듈의 계약은 유지합니다. 이번 패치는 엔진 결과를 표시하는 UI 동기화 계층의 반복 갱신을 안정화한 변경입니다.
