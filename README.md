# AI 쇼츠 제작 스튜디오 v1.2.6

음악이나 영상을 브라우저 안에서 분석하고, 하이라이트 추천부터 세로 미리보기와 저장까지 이어주는 로컬 웹 스튜디오입니다.

## v1.2.6 핵심 패치

- PC·모바일 작업 화면의 테두리, 표면, 그림자, 간격을 하나의 UI 체계로 통일했습니다.
- 중첩 카드와 과한 글로우를 줄이고 여백과 명암으로 작업 계층을 구분합니다.
- PC 메뉴바를 하단 중앙의 8개 항목 플로팅 도구막대로 정리했습니다.
- 모바일 메뉴바는 4열×2행을 유지하면서 높이와 비활성 대비를 개선했습니다.
- 모바일 상단과 4단계 안내를 압축하고 문장 줄바꿈을 정리했습니다.
- 입력·버튼·업로드·후보·파형·스크롤바의 디자인을 일관되게 맞췄습니다.
- v1.2.5의 39개 핵심 스크립트와 단계형 UI 로딩 구조는 그대로 유지합니다.

## 화면별 사용 흐름

### PC

상단의 `새 쇼츠 만들기` 또는 작업실의 `파일 열기`로 원본을 선택합니다. 진행 단계가 바뀌면 메뉴바 활성 상태와 현재 작업 카드가 함께 이동합니다. 첫 화면에서는 불러오기·추천·세로 미리보기·후보·파형을 동시에 확인할 수 있습니다.

### 모바일

상단에서 버전과 모바일·PC 호환 정보를 확인한 뒤 하단 메뉴바의 `파일 열기`로 시작합니다. 본문에는 중복 파일 선택 버튼 대신 4단계 제작 순서만 표시합니다.

## 모듈형 엔진

오디오 특징 추출, 영상 모션 분석, 자동 컷, 추천 점수, 렌더 품질과 안정성 감사를 독립 모듈로 유지합니다. v1.2.6은 엔진 계약을 변경하지 않고 UI 표현 계층만 정리했습니다.

## 단계별 UI 로딩

| 단계 | 준비되는 기능 | 시점 |
|---|---|---|
| Core | 앱 상태, 분석·추천·렌더 핵심, 메뉴 내비게이션 | 첫 화면 |
| Shell | 흐름 보정, UX 동기화, 작업 공간 안정화, 세션 연속성 | 유휴 시간 또는 첫 접근 |
| Editing | 범위 조절, 후보 비교·고정, 저장 준비, 품질 계획 | 후보·미리보기·편집 접근 |
| Export | 저장 완료 센터 | 내보내기 접근 |

현재 단계는 개발자 도구에서 확인할 수 있습니다.

```js
document.body.dataset.hydrationReady
AIShortsStagedUiLoader.readyPhases()
```

## 실행

```bash
npm run serve
```

브라우저에서 `http://localhost:8080`을 엽니다. `index.html`을 직접 여는 것보다 로컬 서버 실행을 권장합니다.
