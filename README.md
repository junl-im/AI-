# AI 쇼츠 제작 스튜디오 v1.6.2

브라우저에서 미디어 분석, 쇼츠 후보 추천, 자막 편집, 세로 영상 렌더링을 수행하는 로컬 우선 제작 도구입니다. v1.6.2는 저장소 상태를 페이지 최하단 지원 영역으로 이동하고, Local AI를 제작 흐름 안의 접이식 작업대로 재배치해 화면 우선순위와 반응형 정렬을 개선합니다.

## 핵심 기능

기존 **모듈형 엔진** 계약 위에서 다음 기능이 동작합니다.

- 오디오·영상 움직임 분석과 쇼츠 후보 추천
- 자동 컷, 후보 고정, 자막·SRT 편집, 9:16 렌더 큐
- 옵션 signature 기반 분석 캐시와 namespace 선택 정리
- 세션 순환 백업과 중요 백업 내보내기·가져오기
- 서비스워커 앱 셸 SHA-256 무결성 검사와 복구
- Ollama·llama.cpp 구조화 카피 생성
- whisper.cpp·OpenAI 호환 로컬 전사 및 SRT 생성
- Local AI 모델 digest 고정, 직렬 작업 큐, 취소·시간 제한·진행률

## v1.6.2 UI/UX 배치

- 저장 공간·오프라인 상태 요약은 전체 제작 작업실 뒤 페이지 최하단에 배치합니다.
- 정상 상태에서는 시각적 우선순위를 낮추고, 정리·복구가 필요해질 때만 한 번 자동 이동·강조합니다.
- `namespace`, `signature`, 캐시 삭제, 셸 감사 기능은 명시적으로 여는 고급 진단 창 안에 유지합니다.
- Local AI는 프로젝트·카피 흐름과 연결되는 전체 폭 접이식 작업대로 배치하며 기본 상태는 닫힘입니다.
- 미리보기·파형 집중 모드에서는 선택형 Local AI 영역을 숨겨 핵심 편집 공간을 보존합니다.
- 데스크톱·태블릿·모바일에서 가로 overflow 없이 정렬됩니다.

## 로컬 AI 원칙

1. 페이지를 열기만 해서는 AI 서버에 요청하지 않습니다.
2. 연결 확인·카피 생성·전사는 사용자의 명시적 동작 뒤에만 실행됩니다.
3. 기본 설정에서는 loopback 주소만 허용합니다.
4. 외부 CDN, 원격 모델, API 키는 배포본에 포함하지 않습니다.
5. Local AI가 없거나 실패해도 기존 분석·편집·렌더 기능은 계속 동작합니다.
6. Local AI 모듈은 해당 영역을 처음 사용할 때만 단계 적재합니다.

## 실행

```bash
npm run serve
```

브라우저에서 `http://127.0.0.1:8080`을 엽니다. Local AI를 사용하려면 별도로 설치한 서버를 loopback 주소에 실행하고 이 앱 origin을 허용하도록 CORS를 설정해야 합니다.

## 검수

```bash
npm test
```

v1.6.2는 자동 검사 221개, 일반 화면 4개 viewport 감사, 저장소 자동 이동 감사, Local AI 배치 전용 데스크톱·모바일 감사, CSS ownership, 서비스워커 lifecycle, GPU/media capability, 프로세스 메모리 보조 감사를 수행합니다.

## 패키징

```bash
npm run package:full
PATCH_BASE_REF=e7c9016 PATCH_FROM_VERSION=1.6.1 npm run package:patch
```

전체 ZIP은 실행·문서·QA 파일을 포함하고 `.git`, `node_modules`, 중첩 `dist`를 제외합니다. 패치 ZIP은 v1.6.1 기준 변경·신규 파일만 포함합니다.

## 알려진 제한

- AI 모델과 로컬 서버 실행 파일은 배포본에 포함되지 않습니다.
- 실기기 Safari·Samsung Internet, 물리 GPU·NPU는 별도 검증이 필요합니다.
- 브라우저 보안 정책 때문에 Local AI 서버의 CORS 설정이 필요할 수 있습니다.
- 프로세스 메모리 수치는 headless Chromium 환경의 보조 지표입니다.
