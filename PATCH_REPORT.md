# AI Shorts Studio v1.5.26 패치 보고서

## 1. 진행 내용

### 영구 분석 캐시 용량 자동 조정·선택 삭제

- `navigator.storage.estimate()` 사용량을 기준으로 영구 캐시 정책을 자동 조정합니다.
- 저장소 정상 상태는 최대 8개·16MiB, 경고 상태는 기본 한도의 60%, 위험 상태는 최소 2개·4MiB로 제한합니다.
- quota 오류가 발생하면 만료 항목과 오래된 LRU 항목을 적극 정리한 뒤 쓰기를 한 번 재시도합니다.
- 정책 상태·사용률·정리·오류·선택 삭제 횟수를 분석 진단 통계에 기록합니다.
- 영구 캐시 목록에는 원시 키나 파일명 대신 비식별 토큰, 예상 바이트, 생성 시각, 최근 사용 시각만 표시합니다.
- 진단 화면에서 선택한 영구 캐시 항목 하나만 삭제하거나 전체 분석 캐시를 정리할 수 있습니다.

### 서비스워커 감사 이력·자산별 백오프

- 주기 무결성 검사 결과를 최신 순으로 최대 40개 보관합니다.
- 동일 자산이 반복 실패하면 5분부터 시작해 최대 6시간까지 지수 백오프를 적용합니다.
- 백오프 중인 자산은 다음 표본에서 건너뛰고, 정상 또는 복구 성공 시 해당 백오프를 즉시 해제합니다.
- 브라우저가 온라인으로 복귀하면 미뤄진 무결성 감사를 자동 재개합니다.
- 최근 표본 검사, 복구 결과, 감사 이력, 자산별 백오프, 이전 정상 캐시 보존 상태를 JSON으로 내보낼 수 있습니다.
- 서비스워커 상태 보고서가 아직 없는 초기 환경에서 불필요한 전체 캐시 검사를 수행하던 경계 사례를 수정했습니다.

### 중요 세션 백업·복원 전 미리보기

- 기본 세션·순환 백업과 별도로 회전 정리되지 않는 `protected` 중요 백업 슬롯을 추가했습니다.
- 사용자가 선택한 정상 스냅샷을 중요 백업으로 복사하거나 보호 해제할 수 있습니다.
- 자동 복구 시 중요 백업을 순환 백업보다 먼저 검사합니다.
- 복원 전 프로젝트 제목, 추천 후보 수, 자막 수, 선택 구간, 압축 여부·절감률을 미리 확인할 수 있습니다.
- 중요 백업 생성·해제·복원 성공·실패는 기존 복구 이력에 함께 기록됩니다.

## 3. 적용 방법

- 전체본 ZIP은 독립적으로 압축을 풀어 실행합니다.
- 덮어쓰기 패치 ZIP은 **v1.5.25 전체본의 최상위 폴더**에 압축을 풀고 기존 파일을 덮어씁니다.
- 삭제가 필요한 파일은 없습니다.

## 4. 알려진 제한

- `navigator.storage.estimate()` 값은 브라우저가 제공하는 추정치이며 지원되지 않는 환경에서는 기본 한도를 사용합니다.
- 항목별 캐시 크기는 JSON 직렬화 기반 예상치이며 브라우저 내부 IndexedDB 오버헤드와 정확히 같지 않을 수 있습니다.
- 서비스워커 백오프 상태는 현재 앱 셸 캐시의 설치 보고서에 저장되며 브라우저가 캐시를 지우면 함께 초기화됩니다.
- 중요 백업은 회전 정리에서 보호되지만 사용자가 앱 데이터 전체 삭제 또는 중요 보호 해제를 실행하면 제거됩니다.
- 장시간 영상 전체 재감사는 미디어 소유 경로가 변경되지 않아 v1.5.24 검증 근거를 상속했습니다.
- 모바일 Safari·Samsung Internet·물리 GPU 검증은 실제 기기가 필요합니다.
