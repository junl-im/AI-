# v1.6.4 Patch Report

## 문제

저장소 문제 발생 시 사용자는 페이지 최하단 상태 영역으로 자동 이동했지만, 해결 완료 후 원래 작업 흐름으로 돌아가는 경로가 없었습니다. 또한 저장소 정리 확인창은 안전 안내만 제공하고 실제 정리 대상 범위를 보여주지 않았습니다.

## 해결

- 자동 이동으로 시작된 복구 흐름을 별도 상태로 추적
- 자동 해결이 성공하고 건강 상태가 정상으로 바뀐 경우에만 최상단 복귀
- 취소·실패·미해결 상태에서는 하단 유지
- 복귀 시 작업 시작 버튼 포커스 및 중복 복귀 방지
- 삭제 전 오래된 백업 수·예상 로컬 용량·이전 오프라인 캐시 수 미리보기
- 미리보기 API를 읽기 전용으로 분리

## 결과

- 하단 이동과 상단 복귀가 하나의 완결된 사용자 흐름이 됨
- 사용자가 정리 영향을 확인한 뒤 실행 또는 취소 가능
- 기존 작업실 배치, 단계 집중, Local AI 지연 적재, 오프라인 무결성 계약 유지
- 등록 자동 검사 224개 통과
