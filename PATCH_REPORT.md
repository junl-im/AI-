# AI Shorts Studio v1.5.25 패치 보고서

## 1. 진행 내용

### 분석 캐시 진단·영구 계층

- 기존 메모리 LRU 캐시 뒤에 선택적 IndexedDB 영구 캐시를 추가했습니다.
- 영구 캐시는 엔진 버전 namespace, 7일 TTL, 최대 8개 항목, 총 16MiB 제한을 적용합니다.
- 만료 항목을 우선 정리하고 마지막 사용 시각 기준 LRU로 항목 수·용량 상한을 유지합니다.
- IndexedDB 미지원·차단·quota 오류가 발생해도 분석 실패로 전파하지 않고 메모리 캐시 또는 새 분석으로 자동 폴백합니다.
- 캐시 적중·미적중·만료·퇴출·정리·쓰기 실패와 지문 처리 시간·읽은 바이트를 진단에 기록합니다.
- 진단 JSON은 파일명·로컬 경로·원본 미디어·원시 캐시 키·분석 결과 본문을 제외하고 비식별 토큰과 집계값만 포함합니다.
- 저장소 상태 화면에 분석 진단 저장과 메모리·영구 캐시 정리 기능을 연결했습니다.

### 세션 백업 선택 복원

- 기본 세션과 각 순환 백업을 개별 디코딩·체크섬·프로젝트 스키마 검증해 복원 시점 목록으로 제공합니다.
- 백업별 저장 시각, 압축 여부·절감률, 정상·손상 상태를 표시합니다.
- 사용자가 원하는 정상 스냅샷을 직접 선택해 복원할 수 있습니다.
- 손상된 백업은 목록에는 표시하지만 선택·복원할 수 없도록 차단합니다.
- 선택 복원 성공·실패를 기존 복구 이력에 연결했습니다.

### 서비스워커 순환 무결성 감사

- 페이지 초기 30초 뒤, 이후 기본 15분 간격으로 유휴 무결성 검사를 예약합니다.
- 한 주기에 앱 셸 12개 파일을 검사하고 cursor를 저장해 다음 주기에서 이어서 전체 셸을 순환합니다.
- 페이지가 숨겨졌거나 오프라인이면 검사를 연기하고, 지원 환경에서는 `requestIdleCallback` 안에서 실행합니다.
- 표본에서 누락·HTTP 비정상·SHA-256 콘텐츠 손상이 발견된 파일만 재다운로드하고 다시 해시 검증합니다.
- 진단 화면에 수동 `셸 표본 검사` 기능과 최근 검사 상태를 추가했습니다.
- 기존 전체 셸 수동 복구, 설치 검증, 새 캐시 실패 시 이전 정상 캐시 보존 정책은 유지합니다.

### 회귀 검사 확장

- `session_backup_selection_smoke.js`
- `service_worker_periodic_integrity_smoke.js`

## 3. 적용 방법

- 전체본 ZIP은 독립적으로 압축을 풀어 실행합니다.
- 덮어쓰기 패치 ZIP은 **v1.5.24 전체본의 최상위 폴더**에 압축을 풀고 기존 파일을 덮어씁니다.
- 삭제가 필요한 파일은 없습니다.

## 4. 알려진 제한

- IndexedDB 영구 분석 캐시는 브라우저 정책과 사용 가능한 quota에 따라 저장되지 않을 수 있으며, 이 경우 메모리 캐시와 정상 분석 흐름으로 폴백합니다.
- 영구 캐시에는 분석 결과 일부가 브라우저 로컬에 저장됩니다. 서버 전송은 없으며 진단 내보내기에는 결과 본문·파일명·경로를 포함하지 않습니다.
- 서비스워커 순환 검사는 한 번에 12개 파일만 검사하므로 전체 앱 셸을 모두 확인하려면 여러 검사 주기가 필요합니다.
- 숨김 탭·오프라인 상태에서는 자동 무결성 검사가 미뤄집니다.
- 장시간 영상 전체 재감사는 미디어 소유 경로가 변경되지 않아 v1.5.24 검증 근거를 상속했습니다.
- 모바일 Safari·Samsung Internet·물리 GPU 검증은 실제 기기가 필요합니다.
