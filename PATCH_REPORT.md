# PATCH REPORT v1.6.1

## 목표

저장소·오프라인 진단 기능은 유지하되, 일반 사용자에게 내부 캐시 구조와 파괴적 관리 버튼이 항상 노출되던 UX를 수정했습니다. 정상 상태의 기본 화면은 작고 이해하기 쉬워야 하며, 전문 도구는 사용자가 명시적으로 진입한 고급 진단 안에서만 실행되어야 합니다.

## 사전 진단

- 기본 화면에 namespace, signature, 셸 감사, 캐시 삭제 버튼이 한꺼번에 노출됐습니다.
- 정상 상태에서도 넓은 영역을 차지해 핵심 제작 흐름을 방해했습니다.
- 분석 캐시 목록과 유지보수 스냅샷을 기본 설치 시점에 조회했습니다.
- 전체 캐시·오래된 저장소·감사 이력 정리에 별도 확인 계층이 부족했습니다.
- 작업 트리에 Local AI 직접 스크립트 3개가 다시 들어와 staged loading 계약을 깨뜨릴 가능성이 있었습니다.

## 구현

### 사용자용 요약

- 저장소·오프라인 상태, 사용량, 필요한 경우의 자동 정리만 표시합니다.
- 정상 상태에서는 자동 정리 버튼을 숨깁니다.
- 기술 용어와 관리 도구는 일반 요약 마크업에서 제외했습니다.

### 고급 진단 modal

- 명시적 버튼 동작 뒤에만 열립니다.
- 기존 namespace·signature·캐시·셸 무결성·감사 이력 기능을 modal 내부로 이동했습니다.
- 모바일에서는 화면 전체를 사용하고 일반 문서 높이를 늘리지 않습니다.
- 키보드 Escape, 포커스 trap, 닫은 뒤 진입 버튼 포커스 복귀를 지원합니다.

### 지연 진단

- 초기 install은 가벼운 저장소·세션·서비스워커 상태만 읽습니다.
- 분석 캐시 상세 목록과 namespace·signature 스냅샷은 고급 진단을 열 때 조회합니다.
- 고급 창을 다시 그릴 때 기존 유지보수 스냅샷을 재사용합니다.

### 위험 작업 확인

- 캐시·namespace·signature·오래된 저장소·감사 이력 정리에 확인 modal을 추가했습니다.
- 현재 프로젝트 원본과 편집 데이터가 삭제되지 않는다는 설명을 제공합니다.
- 취소 경로에서는 삭제 함수를 호출하지 않습니다.

### 성능 회귀 방지

- Local AI 직접 스크립트 재삽입을 제거했습니다.
- staged loader를 통한 첫 사용 시 적재와 직접 실행 스크립트 49개 예산을 복구했습니다.

## 검증

- 정적 visibility 검사로 일반 요약의 기술 용어·위험 버튼 비노출 확인
- 전용 Playwright 감사로 데스크톱·모바일 modal, overflow, 포커스, 확인·취소, 조건부 자동 정리 확인
- 기존 4개 viewport 일반 감사 오류·console error·가로 overflow 0
- CSS ownership 충돌·동일값 중복·shadowed declaration 0
- 서비스워커 lifecycle, GPU/media, 프로세스 메모리, 구조 우선순위 검사 통과
- 등록 자동 검사 **219개**

## 개인정보와 데이터 영향

- 새 네트워크 요청은 추가하지 않았습니다.
- 일반 화면에는 원시 캐시 키, namespace 원문, 파일명, 경로가 노출되지 않습니다.
- 고급 진단도 기존 비식별 토큰 계약을 유지합니다.
- 캐시 정리는 프로젝트 원본·편집 상태를 삭제하지 않으며 분석 결과만 재생성 대상이 될 수 있습니다.

## 남은 제한

- 브라우저 기본 `<dialog>`가 아니라 기존 호환성을 위한 custom modal입니다.
- 실제 모바일 Safari·Samsung Internet의 보조기술 조합은 별도 실기기 확인이 필요합니다.
- headless Chromium의 메모리·GPU 수치는 물리 장치 성능을 보장하지 않습니다.
