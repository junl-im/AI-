# AI Shorts Studio v1.5.24 패치 보고서

## 1. 진행 내용

### 세션 백업 압축·동적 보존

- 기본 세션 JSON은 기존 호환성을 위해 평문으로 유지하고 순환 백업에만 압축을 적용합니다.
- `AISSB1:` 압축 봉투에 LZW16 데이터, UTF-8 원본 크기, 문자 수, FNV-1a 체크섬을 저장합니다.
- 최소 4% 이상 절감될 때만 압축하며 효과가 낮은 데이터는 평문 백업으로 저장합니다.
- 저장소 상태가 정상일 때 최대 3개, 경고일 때 2개, 위험일 때 1개로 백업 보존 개수를 자동 조정합니다.
- 상태 화면에서 압축 백업 수와 총 절감률을 확인할 수 있습니다.

### 복구 이력·진단 내보내기

- 기본 세션 로딩 실패, 백업 복원 성공·실패, 실제 작업 복원 성공·실패, 기록 삭제를 최대 20개 이력으로 저장합니다.
- 압축 백업 복원 시 바이트 수·문자 수·체크섬을 확인한 뒤 프로젝트 스키마 검증을 수행합니다.
- 복구 진단 JSON에는 백업 압축 방식·크기·절감률·체크섬과 복구 이력만 포함하며 프로젝트 원문은 포함하지 않습니다.
- 세션 패널에 `복구 진단` 내려받기 기능을 추가했습니다.

### 서비스워커 SHA-256 콘텐츠 무결성·롤백

- 빌드 도구가 앱 셸 119개 파일의 SHA-256을 `asset-integrity.json`으로 생성합니다.
- manifest 자체 SHA-256은 서비스워커에 고정해 무결성 기준 파일도 검증합니다.
- 설치·상태 확인·수동 복구·활성화 단계에서 캐시 응답 본문을 해시 비교합니다.
- 누락, HTTP 비정상, 본문 손상을 각각 분류하고 손상 항목은 최대 2회 다시 내려받아 재검증합니다.
- 핵심 셸 검증이 실패하면 새 서비스워커 활성화를 중단하고 이전 정상 캐시를 삭제하지 않습니다.
- 진단 상태에 hash verified 수, 손상 파일, manifest 상태, 롤백 보존 캐시를 표시합니다.

### 회귀 검사 확장

- `session_recovery_diagnostics_smoke.js`
- `service_worker_content_integrity_smoke.js`

## 3. 적용 방법

- 전체본 ZIP은 독립적으로 압축을 풀어 실행합니다.
- 덮어쓰기 패치 ZIP은 **v1.5.23 전체본의 최상위 폴더**에 압축을 풀고 기존 파일을 덮어씁니다.
- 삭제가 필요한 파일은 없습니다.

## 4. 알려진 제한

- 내장 LZW16은 동기 localStorage 흐름과 무의존성을 위한 방식으로 Brotli·gzip과 같은 최고 압축률을 목표로 하지 않습니다.
- FNV-1a 체크섬은 우발적 손상 탐지용이며 암호학적 인증이나 서명을 제공하지 않습니다.
- 서비스워커 콘텐츠 해시는 Web Crypto 지원 환경에서 검증합니다. 미지원 환경은 기존 존재·HTTP 상태 검사로 폴백합니다.
- 장시간 영상 전체 재감사는 미디어 경로가 변경되지 않아 v1.5.23 검증 근거를 상속했습니다.
- 모바일 Safari·Samsung Internet·물리 GPU 검증은 실제 기기가 필요합니다.
